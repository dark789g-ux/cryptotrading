# 「组合模拟」使用指南

> 面向使用者的操作手册。页面:侧边栏 →「组合级模拟器」(路由 `/portfolio-sim`,需管理员)。
> 设计/实现细节见 [组合模拟三期改造](../../docs/superpowers/specs/2026-06-14-portfolio-sim-engine-upgrade-design/index.md)。
> **2026-06-15 变更**：原「内联新建信号源」入口已移除,信号方案统一在「信号前向统计」创建并运行(那里现在还能给方案配**迷你回测**,见 [信号前向统计使用指南](./信号前向统计使用指南.md));组合源选择器收成「选已有方案 / 手填 uuid」两态。详见 [signal_test 迷你回测升级](../../docs/superpowers/specs/2026-06-14-signaltest-minibacktest-and-job-drafts-design/index.md)。

## 1. 这是什么、什么时候用

「组合模拟」把一个**已有信号策略的逐笔交易**(来自「信号前向统计 / signal-stats」的一次 run),放进一个**共享资金池、有仓位上限、有真实交易成本**的组合里逐日回放,回答一个单策略胜率/凯利答不了的问题:

> **这个策略在真实容量约束和成本下,组合层面到底赚不赚钱?**

典型场景:
- 某策略「逐笔前向胜率/凯利」很漂亮,但一天触发上百个信号,组合只吃得下几十个——**被容量绞杀后还剩多少 edge?**
- 同日候选超额时,**按什么挑那几十个**(信号质量 / 市值 / 破位位置)对最终收益影响多大?
- 多个策略合成一个组合,各自分多少仓、总敞口怎么控?

**信号源**:每个源消费一次 signal_test_run 的逐笔(`signal_test_trade`),**不重新扫 K 线**。在「新建方案」里给源指定信号有**两种方式**(详见 §3.1):选已有方案的一次历史 run、或手填 run id。要用一个全新信号方案,先去「信号前向统计」把它建好并运行,再回来用「选已有方案」选它的 run。

---

## 2. 快速上手(最小步骤)

```
组合级模拟器 →「新建方案」
   ① 填方案名 + 初始资金(默认 100 万)
   ② 加一个信号源 → 先选「来源方式」(二选一,详见 §3.1):
        · 选已有方案 → 再选一次历史 run(仅已完成可选,默认最新)
        · 手填 uuid  → 粘 signal_test_run 的 id
      再填 标签 + 单票仓位 + 最大持仓数
   ③ 选成本档(现实 / 乐观 / 保守 / 零成本)
   ④ (可选)排序 / 动态仓位 / 熔断
   ⑤ 保存 → 列表里点「运行」
        │
        ▼
   等进度跑完(loading → replaying → writing)→ 看详情:净值曲线 + 汇总指标 + 逐笔明细
```

> 两种来源最终都只落一个**已完成、逐笔 > 0** 的 `signal_test_run` id。要用全新方案,先去「信号前向统计」建+跑,跑完它的 run 才会出现在「选已有方案」的历史 run 下拉里。

## 3. 配置详解

### 3.1 信号源(每个方案 1~5 个)

**来源方式(二选一)**——每个源行先在顶部「来源方式」里选怎么指定信号,两种最终都只落一个 `runId`,后续配置(仓位/排序/sizing)完全一样:

| 来源方式 | 怎么用 | 适合 |
|---|---|---|
| **选已有方案** | 选一个信号方案 → 再选它的一次**历史 run**(仅「已完成」可选,默认选最新已完成);下方显该源的只读条件摘要(买入条件数 / 出场 / 范围 / 区间 / 样本 / 胜率)帮你确认选对 | 复用已有方案;**想用某次旧 run 而非最新那次**也能精确选 |
| **手填 uuid** | 直接粘 `signal_test_run` 的 id | 你已知道确切 run id(脚本产出 / 跨页面拿到) |

> 「选已有方案」默认就是该方案最新已完成 run(等同旧版「选方案」行为);要换历史 run 才需动第二个下拉。
>
> **要一个全新信号方案?** 去侧边栏「信号前向统计」新建并运行(那里现在是 7-tab 表单,还能给方案配迷你回测),跑完回到这里用「选已有方案」选它的 run。某方案选了却显示「无可用 completed run」,就是它还没跑过——去那边点「运行」。

每个源行的字段:

| 字段 | 含义 | 建议 |
|---|---|---|
| **runId** | 信号源(signal_test_run 的 id),逐笔交易来自它 | 由上面「来源方式」产生,不必手记 |
| **label** | 该源的人类标签(组内唯一) | 如 `V10-E7` |
| **positionRatio** | 单票占 NAV 的比例,`alloc = positionRatio × 上一日NAV` | 0.02~0.05 常见;越小越分散 |
| **maxPositions** | 该源最大同时在仓数;`null`=不限 | 容量约束核心,如 30/50 |
| **exposureCap** | 该源总敞口上限(占 NAV);`null`=不限 | 如 0.95 |

> 多源 = 组合多个策略。开仓时按源在方案里的顺序逐个处理,各用各自的仓位/约束/排序。

### 3.2 排序:同日候选超额时「挑哪几个」(★最影响结果)

当某天信号数 > 剩余仓位,引擎按**排序**决定谁先进。三种形态:

- **不排序(none)**:按 ts_code 字母序——等于「盲挑」,容易逆向选股。
- **单因子**:按某一个因子值排(升/降序)。
- **composite 多因子加权(推荐)**:多个因子各自独立排名 → `分数 = n − 名次` → 乘权重累加 → 综合分降序。同值/缺值并列同分,结果确定可复现。

可选的 **9 个因子**(历史可回测的优先用):

| 因子 | 含义 | 方向(默认) | 历史可回测 |
|---|---|---|---|
| `risk_reward` | 盈亏比,越高越优 | desc | ✅ |
| `momentum_60` | 动量 (close−ma60)/atr,越强越优 | desc | ✅ |
| `pos_60` / `pos_120` | 60/120 日价格位置 | asc(越低越回踩) | ✅ |
| `close_ma60_ratio` | close/ma60 | asc | ✅ |
| `vol_ratio_60` / `vol_ratio_120` | 量比(缩量优先) | asc | ✅ |
| `circ_mv` | 流通市值(小市值优先,非质量) | asc | ✅ |
| `ml_score` | 模型分 P(涨)−P(跌) | desc | ⚠️ **仅前向** |

> ⚠️ **`ml_score` 历史只覆盖 2 天**(每日推理上线后才有),拿它给多年历史回测排序会几乎全空、退化成字母序。**仅用于前向/上线**,前端选项已灰字提示;别用它做历史回测对比。

### 3.3 动态仓位 sizing(可选,默认 fixed)

决定每笔实际下多少(在 positionRatio 基线上乘一个系数,总敞口仍受 exposureCap/仓位数卡死,不放大杠杆):

- **fixed**(默认):`alloc = positionRatio × NAV`,固定比例。
- **signal_weighted**:好信号多配、差信号少配。`乘子 = floorMult + (capMult−floorMult)×日内质量分位`(默认 floor 0.5 / cap 1.5)。**需要配排序**(none 时退化为固定)。
- **source_kelly**:按该源**自身历史 p/b** 算一个静态凯利乘子缩放仓位(一源一标量,不是 per-trade,避免并发杠杆爆炸)。负期望源会被压到 0 仓(信号记 `凯利归零 sized_out`)。

### 3.4 熔断 circuitBreaker(可选,默认全关,账户级)

组合整体风控,两条独立触发线(任一命中即当天停开仓):

- **连亏熔断**:连续亏 N 笔(净收益口径)→ 冷却若干交易日禁开仓;亏损延长、盈利缩短冷却。
- **回撤熔断**:账户净值自峰值回撤 ≥ `drawdownHaltPct`(默认 15%)停开仓,回升到回撤 ≤ `drawdownResumePct`(默认 10%)恢复(滞回,防抖动)。

冻结日的候选记 `连亏熔断 cooldown` / `回撤熔断 drawdown_halt`,仍带逐因子值供复盘。

### 3.5 成本档

| 档 | 佣金 | 过户 | 印花(卖,减半后) | 滑点 |
|---|---|---|---|---|
| 乐观 | 万2.5 | 万0.1 | 万5 | 0 |
| 现实 | 万2.5 | 万0.1 | 万5 | 万5 |
| 保守 | 万2.5 | 万0.1 | 万5 | 千1 |
| 零成本 | 0 | 0 | 0 | 0(对账用) |

> 印花税自动按出场日切档(2023-08-28 前千1、后万5)。高换手策略对滑点档极敏感,先看「现实」再看「保守」。

### 3.6 初始资金 / anchorMode 对账模式

- **initialCapital**:首日 NAV,默认 100 万(只影响绝对额,不影响收益率)。
- **anchorMode(对账模式)**:勾上后**所有约束停用 + 零成本**,每笔信号必成交、且每笔净收益 ≡ 原始 ret。**它不是真实组合回测**,只用来核对「组合引擎重放出的胜率/凯利是否与单策略一致」(详情里有 `anchor_check` 自校验)。真实回测**不要**勾它。

## 4. 跑 + 看结果

**运行**:方案列表点「运行」。进度分三阶段:`loading`(装载逐笔+行情)→ `replaying`(逐日回放)→ `writing`(落库)。同一方案运行中再点会 409(互斥)。刷新页面会自动恢复轮询。

**详情页**看三块:
1. **净值曲线**:组合 NAV 随时间(归一到初始资金=1)。
2. **汇总指标**(怎么读):

| 指标 | 读法 |
|---|---|
| **年化 annualRet** | 最关键。`finalNav<=0` 时为空 |
| 总收益 totalRet / 最大回撤 maxDrawdown | 回撤是峰值回落最大比例(负数) |
| 夏普 sharpe / 卡玛 calmar | 风险调整后收益;calmar=年化/\|回撤\| |
| 日胜率 / 日凯利 | 按日收益序列算 |
| **nTaken / nSkipped** | 成交 / 被拒数——`nSkipped` 高=容量绞杀严重 |
| totalCosts | 全部买卖费用绝对额 |

3. **逐笔明细**(服务端分页/筛选/排序):每笔 `taken/skipped` + 原因。可按状态、源、`skipReason`、买入日筛。
   - **逐因子透明**:展开某行看它当时各因子的原始值 + composite 综合分;`rank值`列按排序类型分支显示(composite 显综合分、单因子显原值)。
   - skip 原因中文:`已持有 / 仓位满 / 敞口超限 / 现金不足 / 连亏熔断 / 回撤熔断 / 凯利归零`。

## 5. 一个真实例子(质量排序救活策略)

用一个真有 edge 但被容量绞杀的 A 股策略(0AMV-Q2 的 V10)做对照,**同容量(50 仓、单票 2%、现实成本),只换排序**:

```
不排序(盲挑)            年化 −1.07%
pos_120 升序(破位优先)   年化 −5.50%   ← 盲挑反而逆向选股
risk_reward 降序          年化 −0.54%
composite(rr+动量)       年化 +0.91%   ✅ 转正
momentum_60 降序          年化 +3.18%   ✅ 最优
```

**结论**:同样的信号、同样的容量和成本,**「按信号质量择优挑」比「盲挑」把年化从 −1.07% 救到 +3.18%**——排序维度直接决定一个被容量绞杀的策略能不能部署。这正是组合模拟最该用来回答的问题。

## 6. 常见坑 / 注意事项

- **源 run 必须已完成**:两种来源方式最终都要一个**成功且逐笔 > 0** 的 run。要用全新方案先去「信号前向统计」建+跑;若某源下拉显示「无可用 completed run」就是它还没跑过,组合点「运行」会被拦(提示源仍在运行)。
- **anchorMode 别用于真实回测**:它零成本无约束,数值无组合意义,只用于对账。
- **ml_score 禁历史回测**:历史仅 2 天,历史区间会退化;只在前向/上线用。
- **容量是第一约束**:`nSkipped` 远大于 `nTaken` 时,先调 `maxPositions/positionRatio/exposureCap` 找到不被绞杀的容量,再谈排序/仓位。
- **成本吞噬**:高换手策略务必看「保守」滑点档,别只看乐观。
- **signal_weighted 要配排序**:不排序时它没有质量分位可用,会退化成固定仓位。

## 7. API 速查(可选,前端已封装)

| 操作 | 接口 |
|---|---|
| 新建方案 | `POST /api/portfolio-sims`(body=`{name, note?, config}`) |
| 触发运行 | `POST /api/portfolio-sims/:id/run` |
| 进度 | `GET /api/portfolio-sims/:id/progress` |
| 详情(含汇总指标) | `GET /api/portfolio-sims/:id` |
| 每日净值 | `GET /api/portfolio-sims/:id/daily` |
| 逐笔明细(分页/筛选/排序) | `GET /api/portfolio-sims/:id/fills?...` |
| 删除 | `DELETE /api/portfolio-sims/:id`(运行中 409) |

`config` 形态:`{ sources:[{runId,label,positionRatio,maxPositions,exposureCap, rankSpec?:{factors:[{factor,weight,dir}]}, sizing?:{mode,...}}], initialCapital, cost:{...5费率}, anchorMode, circuitBreaker? }`。`rankSpec` 存在且非空时接管排序(优先于 legacy `rankField/rankDir`)。
