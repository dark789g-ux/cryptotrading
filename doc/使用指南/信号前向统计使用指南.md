# 「信号前向统计 / 迷你回测」使用指南

> 面向使用者的操作手册。页面:侧边栏 →「信号前向统计」(路由 `/signal-stats`)。
> 设计/实现细节见 [signal_test 迷你回测升级](../../docs/superpowers/specs/2026-06-14-signaltest-minibacktest-and-job-drafts-design/index.md)。

## 1. 这是什么

给一个 A 股买入条件做**逐信号前向统计**:在区间内逐日找出命中买入条件的标的,每个信号按出场规则模拟一笔交易,聚合出**胜率 / 盈亏比 / 凯利 / 样本数**等信号质量指标——回答「这个入场条件本身有没有 edge」。

**2026-06-15 新增:迷你回测层。** 同一次运行,在信号质量统计之上**可选叠加**一层带**资金账户、仓位管理、截面排序、账户级熔断、交易成本**的回测,产出**净值曲线 + 回测指标**(年化/回撤/夏普…)。它复用组合模拟器的引擎,等价于「对这一个信号源跑一次单源组合模拟」。

> **和「组合模拟」的关系**:这里是**单个**信号方案的迷你回测;要把**多个**方案合成一个组合、互相分仓,用[组合模拟](./组合模拟使用指南.md)。两者用同一套资金账户引擎,口径一致。

## 2. 创建与运行是分开的两步

- **保存**:只把方案参数落库,**不跑**。
- **运行**:在方案列表点该行「运行」,才真正计算(异步,有进度条)。

```
信号前向统计 →「新建方案」(7-tab 表单) → 填参数 →「保存」(只落库)
                                                    │
列表里点该方案「运行」 ──────────────────────────────┘
   进度:扫描 → 模拟 →（配了回测才有）资金账户回放 → 写库
   跑完点「详情」:收益率分布 / 逐笔明细 /（配了回测才有）迷你回测净值曲线
```

## 3. 7-tab 表单逐区

表单分 7 个 tab(对齐「新建策略」的形态)。**前 4 项 + 止损与出场决定每笔信号的 `ret`(信号质量层,永远生效)**;**入场排序 / 资金与仓位 / 风控与回测决定迷你回测(资金账户层,只在开了回测开关后生效)**。

| Tab | 内容 | 作用层 |
|---|---|---|
| **基础信息** | 方案名称 / 标的池(全市场 or 指定列表) / 统计区间 | 信号质量层 |
| **基础配置** | `lookback`(phase_lock 初始止损回看窗口);仅 phase_lock 出场模式有效,其它模式显占位 | 信号质量层 |
| **入场信号** | 买入条件(与「策略条件」同款条件构建器) | 信号质量层 |
| **入场排序** | `rankSpec`:同日多信号竞争仓位时按什么挑(单/多因子,见组合模拟指南 §3.2) | 迷你回测 |
| **资金与仓位** | 初始资金 / 单票仓位 positionRatio / 最大持仓 maxPositions / 敞口上限 exposureCap / 动态仓位 sizing | 迷你回测 |
| **止损与出场** | 出场模式 4 选 1:`fixed_n`(持有 N 日) / `strategy`(卖出条件+兜底天数) / `trailing_lock`(波段跟踪止损) / `phase_lock`(两阶段锁定) | 信号质量层 |
| **风控与回测** | **回测层总开关** + 成本档(乐观/现实/保守/零成本) + 双触发熔断 + anchorMode 对账 | 迷你回测 |

> **开关在哪**:迷你回测默认**关**。到「风控与回测」tab 打开回测层总开关,「入场排序 / 资金与仓位」才生效(关着时这两 tab 灰显)。不开 = 纯前向统计,**行为与老版本逐字一致(零漂移)**。

## 4. 跑 + 看结果

详情页(方案跑完后):

- **收益率分布**(永远有):胜率 / 盈亏比 / 凯利 / 样本数 / 单笔收益直方图 / 最差最佳单笔。
- **逐笔明细**(永远有):每笔信号的买入日 / 出场日 / 收益率 / 持有天数 / 出场原因,可排序/筛选/分页。
- **迷你回测**(仅配了回测的 run 才出现):
  - **8 张指标卡**:总收益 / 年化 / 最大回撤 / 夏普 / 卡玛 / 日胜率 / 成交·拒绝(nTaken/nSkipped) / 总成本。
  - **净值曲线**:组合 NAV(归一)随时间 + 回撤带(双 y 轴)。
  - 读法同[组合模拟使用指南](./组合模拟使用指南.md) §4「跑 + 看结果」(同一套引擎/指标)。

> 旧 run(没配回测)的详情**不显示**迷你回测 tab。

## 5. anchorMode 对账模式

「风控与回测」里的 anchorMode 勾上后:**所有约束停用 + 零成本**,每笔信号必成交、每笔净收益 ≡ 原始 ret(`成交/拒绝 = N/0`、`总成本 = 0`)。**它不是真实回测**,只用来核对「资金账户引擎重放出的结果是否与信号统计层一致」。真实回测**不要**勾它。

## 6. 常见坑

- **不开回测开关 = 没有净值曲线**:迷你回测默认关。要看净值曲线,去「风控与回测」tab 打开回测层总开关并配资金/仓位。
- **`lookback` 只对 phase_lock 有意义**:其它出场模式下「基础配置」tab 是占位。
- **排序/仓位仅回测层生效**:不开回测时这些参数不参与信号质量统计(信号统计是等权全取、无资金约束)。
- **ml_score 因子禁历史回测**:历史仅 2 天,见组合模拟指南 §3.2;只在前向/上线用。
- **anchorMode 别用于真实回测**:零成本无约束,数值无组合意义,只用于对账。
- **新方案要作组合源**:这里建好+跑完后,它的 run 才会出现在「组合模拟」的「选已有方案」历史 run 下拉里。

## 7. API 速查(前端已封装)

| 操作 | 接口 |
|---|---|
| 新建方案 | `POST /api/signal-tests`(body 含可选 `backtestConfig`,缺省/`null`=不跑回测) |
| 触发运行 | `POST /api/signal-tests/:id/run` |
| 净值曲线(迷你回测) | `GET /api/signal-tests/:id/runs/:runId/equity` |

`backtestConfig` 形态(扁平单源,字段语义同组合模拟 §7):`{ initialCapital, cost:{...5费率}, anchorMode, positionRatio, maxPositions, exposureCap, rankSpec:{factors:[...]}, sizing:{mode,...}, circuitBreaker? }`。为 `null` 时该次运行只产出信号质量层(零漂移)。
