# 策略回测流程说明

本文档说明加密货币策略回测的完整流程，包括数据加载、回测引擎执行逻辑和报告生成。

---

## 一、整体架构

回测系统采用模块化设计，位于 `backtest/` 目录下：

| 文件 | 职责 |
|------|------|
| `config.py` | 回测参数配置（资金、仓位、路径等） |
| `data.py` | K线数据加载与全局时间轴构建 |
| `models.py` | 持仓(Position)与交易记录(TradeRecord)数据类 |
| `indicators.py` | 近期高低价计算工具 |
| `engine.py` | 回测引擎主循环与信号处理 |
| `report.py` | 统计指标计算与报告生成 |

入口脚本：`backtest_strategy.py`

---

## 二、数据准备

### 2.1 输入数据源

回测基于本地缓存的1小时K线数据：

- **路径**: `cache/1h_klines/{symbol}_1h.csv`
- **格式**: CSV文件，包含以下列：
  - 价格数据: `open`, `high`, `low`, `close`
  - 时间: `open_time` (UTC+8格式: YYYY-MM-DD HH:MM:SS)
  - 指标列: `MA5`, `MA30`, `MA60`, `MA120`, `MA240`, `DIF`, `MACD`, `KDJ.J`
  - 成交量: `10_quote_volume`
  - 止损列: `loss_atr_14` (可选)

### 2.2 数据加载流程 (`data.py`)

1. **扫描目录**: 读取 `cache/1h_klines/` 下所有 `*_1h.csv` 文件
2. **过滤标的**: 排除稳定币等不交易标的（配置于 `EXCLUDED_SYMBOLS`）
3. **列检查**: 确保包含所有必需的指标列
4. **数据清洗**: 数值列转为数值类型，时间列转为 datetime
5. **预热期裁剪**: 丢弃前 `WARMUP_BARS` (240根) K线，确保MA240稳定
6. **回溯缓冲区**: 保留 `LOOKBACK_BUFFER` (50根) 额外K线，用于计算近期高低价
7. **回测区间截取**: 只保留最新的 `MAX_BACKTEST_BARS` 根K线参与回测

### 2.3 全局时间轴构建

取所有交易对回测起始时间的并集，升序排列，形成统一的时间步序列。

## 三、策略逻辑

### 3.1 入场信号条件

当满足以下全部条件时，产生买入信号：

```
close > MA60 AND
MA30 > MA60 AND
MA60 > MA120 AND
close > MA240 AND
KDJ.J < 10
```

**入场执行规则**:
- 信号在当前K线收盘后检测
- 下一根K线开盘价买入
- 按盈亏比排序，优先选择盈亏比最高的标的

### 3.2 仓位管理

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `INITIAL_CAPITAL` | 1,000,000 USDT | 初始资金 |
| `POSITION_RATIO` | 45% | 单仓资金占比 |
| `MAX_POSITIONS` | 2 | 正常最大持仓数 |
| `MIN_OPEN_CASH` | 100 USDT | 开仓最低现金门槛 |

**扩展规则**: 当2个仓位都触发阶段止盈后，允许开第3个仓位。

### 3.3 止损设置

开仓时同时设置止损价，取以下两者的较大值：

1. **ATR止损**: 基于 `loss_atr_14` 列（信号K线的前一根的值）
2. **最大亏损止损**: `entry_price * (1 - MAX_INIT_LOSS)` (默认3%)

### 3.4 止盈策略

**阶段止盈**: 当最高价触及"近期高点"时，卖出一半仓位
- 近期高点计算: 买入点前9根K线的最高价，向前回溯最多50根

## 四、回测引擎执行流程 (`engine.py`)

### 4.1 主循环结构

```python
for ts in timestamps:  # 遍历每个时间步
    1. 执行上一时间步的挂单买入
    2. 处理每个持仓（止盈/止损/收盘规则）
    3. 计算当前持仓市值，记录持仓快照
    4. 扫描入场信号，生成新的挂单
```

### 4.2 单根K线持仓处理流程

对于每个持仓，在同一根K线上按以下优先级处理：

1. **阶段止盈检查**: 若最高价 >= 近期高点，卖出半仓
2. **止损检查**: 若最低价 <= 止损价，全仓止损离场
3. **收盘处理**:
   - 若本根触发了阶段止盈，调节动态止损后检查收盘止损
   - MA5下跌破线出场（优先于MA5上升调节）
   - MA5首次上升后调节动态止损

### 4.3 退出原因分类

| 退出原因 | 说明 |
|----------|------|
| 阶段止盈 | 触及近期高点，半仓出场 |
| ATR止损 / 最大亏损止损 | 触及初始止损价 |
| 阶段止盈后收盘止损 | 半仓止盈后，收盘低于调节后的止损价 |
| MA5下跌破线 | 收盘低于MA5且MA5走平或下跌 |
| MA5上升后止损 | MA5首次上升后，收盘低于调节止损价 |
| 回测结束 | 强制平仓 |

## 五、报告生成 (`report.py`)

### 5.1 统计指标

| 指标 | 计算方式 |
|------|----------|
| 总收益率 | `(最终净值 - 初始资金) / 初始资金 * 100%` |
| 最大回撤 | 净值峰值到谷底的最大跌幅百分比 |
| 夏普率 | 年化夏普率，基于小时收益率 |
| 胜率 | 盈利完整交易次数 / 总完整交易次数 |
| 平均持仓周期 | 所有完整交易的持仓时间平均值 |

### 5.2 输出文件

运行回测后，在 `backtest_results/{run_id}/` 目录生成：

| 文件 | 内容 |
|------|------|
| `trades.csv` | 每笔交易的详细记录 |
| `portfolio.csv` | 净值曲线时间序列 |
| `report_data.json` | 前端报告所需的完整数据 |

## 六、使用方式

### 6.1 运行回测

```powershell
python backtest_strategy.py
```

### 6.2 查看报告

```powershell
python serve_report.py
# 浏览器访问 http://localhost:8888
```

---

## 七、关键配置参数

可在 `backtest/config.py` 中调整：

```python
INITIAL_CAPITAL = 1000000.0   # 初始资金
POSITION_RATIO = 0.45          # 单仓占比
MAX_POSITIONS = 2              # 最大持仓数
WARMUP_BARS = 240              # 指标预热期
MAX_BACKTEST_BARS = 1000       # 回测K线数 (0=不限制)
RECENT_WINDOW = 9              # 近期高低价窗口
LOOKBACK_BUFFER = 50           # 回溯缓冲区
MAX_INIT_LOSS = 0.03           # 最大初始止损比例
```
