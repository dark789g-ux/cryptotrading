# Edit 工具编辑中文时产生乱码

## 背景

2026-04-13 更新 AGENTS.md 时，使用 Edit 工具对含中文的行进行替换，替换完成后发现相邻中文字符被截断为 `��`、`���` 等乱码（mojibake）。共 4 处：`详见` → `详��`、`每个` → `���个`、`仅导出` → `���导出`、`返回/字段/修复` → `��回/字��/修���`。

## 结论

Edit 工具替换中文内容时，`old_string` 边界可能切断相邻的多字节 UTF-8 字符，导致乱码。编辑含中文的文件后**必须检查**结果。

## 详情

### 复现条件

- 文件包含中文（多字节 UTF-8 编码）
- `old_string` 的起止位置恰好位于某个中文字符的 UTF-8 字节序列中间
- 尤其在连续多次 Edit 同一文件时更易触发

### 预防措施

1. **编辑后立即检查**：对含中文的文件执行 Edit 后，Read 相关行确认无乱码
2. **扩大匹配范围**：`old_string` 尽量包含完整的中文词组，不要在汉字中间截断
3. **优先用 Write 整段重写**：对中文密集的文件（如 AGENTS.md、文档），整段 Write 比逐行 Edit 更安全
