# 在云服务器上为其他电脑开通 SSH 登录

本文说明：当有多台电脑（或新电脑）需要 SSH 登录同一台云服务器时，如何在**服务器端**正确登记密钥，以及每台电脑上应保留什么。

---

## 先弄清：服务器上放的是「公钥」，不是「私钥」

| 在哪台机器 | 保留什么 | 能否外传 |
|-----------|---------|---------|
| **你的电脑** | **私钥**（如 `id_ed25519`，无 `.pub` 后缀） | **不能**，绝不复制到服务器或聊天工具 |
| **云服务器** | **公钥**（`id_ed25519.pub` 的内容） | 可以，写入 `authorized_keys` 即表示「信任用对应私钥登录的人」 |

一句话：**私钥像家门钥匙只在自己兜里；公钥像门禁卡号，可以交给物业（服务器）登记。**

---

## 流程概览

1. 在**需要登录的那台电脑**上生成密钥对（若还没有）。
2. 从该电脑复制 **公钥** 的**一整行**文本。
3. 登录云服务器，把这一行追加到对应用户目录下的 `~/.ssh/authorized_keys`。
4. 设置好目录与文件权限。
5. 在该电脑上执行 `ssh 用户@服务器IP` 验证免密登录。

---

## 一、在「新电脑」上准备公钥

### 1. 检查是否已有密钥

**Windows（PowerShell）**：

```powershell
Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub
```

**Linux / macOS**：

```bash
cat ~/.ssh/id_ed25519.pub
```

若提示文件不存在，则生成一对新密钥：

```bash
ssh-keygen -t ed25519 -C "你的邮箱或备注"
```

一路回车即可（也可为私钥设置密码短语，更安全）。

### 2. 复制公钥全文

仍只操作 **`.pub` 文件**，内容形如一行：

```text
ssh-ed25519 AAAA...很长... 备注
```

或旧格式：

```text
ssh-rsa AAAA...很长... 备注
```

整行复制，**不要换行、不要漏字**。

---

## 二、在云服务器上添加公钥

使用已有方式登录服务器（控制台、旧电脑 SSH、或临时密码登录），以要开通登录的 **Linux 用户**为准（例如 `root` 或 `ubuntu`）。下文以用户主目录 `~` 为例。

### 1. 创建目录并设置权限

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
```

### 2. 编辑 `authorized_keys`

```bash
nano ~/.ssh/authorized_keys
```

- 若文件中已有其他机器的公钥，**在末尾新起一行**，粘贴刚复制的那一行。
- **每台电脑一行**；不要在一行里粘多段密钥。

### 3. 保存并退出（nano）

| 操作 | 按键 |
|------|------|
| 保存 | `Ctrl + O`，回车确认文件名 |
| 退出 | `Ctrl + X` |

### 4. 设置文件权限

```bash
chmod 600 ~/.ssh/authorized_keys
```

权限过宽时，SSH 会拒绝用密钥登录。

---

## 三、在「新电脑」上验证

```bash
ssh 用户名@服务器公网IP
```

首次会提示确认主机指纹，输入 `yes` 即可。若配置正确，**不应再要求输入服务器登录密码**（除非你给私钥设置了密码短语，那时会提示输入**私钥**的密码短语，这是正常的）。

---

## 四、可选：用 `ssh-copy-id` 自动追加（新电脑能密码登录时）

在**已经能密码 SSH 登录**的前提下，可在新电脑上执行：

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub 用户名@服务器IP
```

按提示输入一次服务器密码，会把本机公钥自动追加到服务器的 `authorized_keys`。

---

## 五、与 VS Code Remote-SSH 的配合

在本机配置 `~/.ssh/config`（Windows 路径一般为 `C:\Users\<用户名>\.ssh\config`），例如：

```ssh-config
Host my-cloud
    HostName 服务器公网IP
    User root
    IdentityFile C:\Users\Lucifer\.ssh\id_ed25519
    IdentitiesOnly yes
```

然后在 VS Code 中：`Remote-SSH: Connect to Host`，选择 `my-cloud`。  
**`IdentityFile` 指向的是本机私钥**，与服务器上的公钥成对。

---

## 六、撤销某台电脑的访问

在服务器上编辑 `~/.ssh/authorized_keys`，**删除对应那一行公钥**，保存后，该私钥即无法再登录此用户。

---

## 七、安全提醒

1. **不要把私钥文件发给任何人，也不要粘贴到服务器或网页。**
2. 公钥可以备份、可以贴到多台服务器；私钥丢失或泄露应重新 `ssh-keygen` 并更换服务器上的对应公钥行。
3. 若使用云厂商控制台「重置密码」等功能，不影响已配置的 SSH 公钥，除非你在服务器上删改了 `authorized_keys`。
