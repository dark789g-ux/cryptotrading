## 3.1 三大 PIT 铁律

| 铁律 | 反例（必翻车） |
|---|---|
| 财务用披露日不用报告期 | 2024Q1 财报 4-28 才公告，4-1 就用 → 偷看未来 |
| 行情用 T 日盘后落 | T 日的 `fwd_5d_ret` 必须 T+5 日才算 |
| 因子窗口不跨越未来 | 60 日动量只用 T-60..T-1，T 日不进窗口 |

## 3.2 三个幽灵 Bug

### Bug 1：幸存者偏差

只用当前仍上市的股票回测历史 → 历史 IC 被严重高估，上线必崩。

**正确做法**：每个交易日还原当时的可交易股票池（`list_date <= 当日 AND (delist_date IS NULL OR delist_date > 当日)`），按日物化为 `tradable_universe` 表。

### Bug 2：复权陷阱

A 股财报、分红、配股都会改变历史价。今天看到的"历史复权价"已经经过未来分红调整，直接用作训练特征 = 偷看未来。

**正确做法**：用后复权价为基准（不随时间变化），但每日的复权因子按 PIT 独立存储；T 日的因子只用 T 日及之前的复权因子。

### Bug 3：财务披露延迟

财报报告期（如 2024Q1 = 2024-03-31）≠ 公告日（如 2024-04-28）。

**正确做法**：合并财务数据时用 `ann_date + 1 个交易日` 作为可用日，且按 backward 方向 merge_asof。

## 3.3 数据质量校验（每次落库必跑）

| 检查项 | 阈值 / 期望 | 不满足时 |
|---|---|---|
| symbol_count | 4800-5200（A 股全市场） | 阻断 |
| duplicate_keys (symbol, trade_date) | = 0 | 阻断 |
| null_close_pct | < 0.05 | 告警 |
| volume_zero_pct | < 0.10 | 告警 |
| extreme_return (\|pct_chg\| > 20) | = 0 | 阻断（必是数据错误） |
| extreme_turnover (> 100) | = 0 | 阻断 |
| fina_lag_days (ann_date - end_date) | 中位数 30-90 天 | 告警 |
| adj_factor_jump (单日变化 > 50%) | < 5 | 告警 |

任一项违规 → 阻断后续训练，人工介入。

## 3.4 因子计算工程化

**模式**：每个因子 = 独立函数 + 单元测试 + **PIT 测试**。

**PIT 测试核心**：对同一因子，用全量历史数据 vs 截止某历史日的子集分别计算，**该历史日的因子值必须严格相等**。任何"用了未来数据"的实现都会在此测试失败。

**性能目标**：单因子全市场 5 年计算 < 30 秒，41 因子并行 < 10 分钟。

**实现建议**：矢量化优先（pandas groupby + rolling），复杂形态因子用 numba 加速。

### 因子分类与计算顺序

| 因子类型 | 计算依赖 | 计算成本 | 例子 |
|---|---|---|---|
| 截面因子 | 仅当日数据 | 极低 | 当日涨幅、换手率 |
| 时序因子 | 历史 N 日窗口 | 中 | 20 日动量、60 日波动率 |
| 形态因子 | 复杂窗口 + 状态机 | **高** | 波段识别、形态匹配 |
| 横截面排序因子 | 当日全市场 | 中 | 行业内 ROE 排名 |
| 基本面因子 | 财务表 + PIT 对齐 | 中 | PE、ROE、毛利率 |
| 资金面因子 | 龙虎榜 / 资金流 | 中 | 主力净流入 5 日累计 |

**计算顺序**：截面 → 时序 → 形态 → 横截面排序（依赖前面的）。

---

## 3.5 因子落库规范

**宽表（推荐）**：落入 `factors.factor_panel`，`(symbol, trade_date)` 为主键 + 41 个因子列 + 元信息（行业代码 / 是否 ST / 是否停牌）。

**版本控制至关重要**：因子计算逻辑改一次 → 字段 `factor_version` 升一位 → 训练时只能用同一 version 的数据。同一 `(symbol, trade_date)` 在不同 version 下的值可共存（联合主键 `(symbol, trade_date, factor_version)`）。

**分区策略**：按月分区，超过 3 年的分区 `DETACH` 后归档。
