## 5.1 GBDT 简介

**GBDT = Gradient Boosting Decision Tree**：用一棵接一棵的小决策树，每棵树拟合**前面所有树的负梯度**（损失函数对当前预测的负方向），最后预测 = 所有树输出之和。

| 维度 | 决策树 | 随机森林 | GBDT |
|---|---|---|---|
| 树数量 | 1 | 几百 | 几百-几千 |
| 训练方式 | 单次 | 并行 bagging | **串行 boosting** |
| 单棵深度 | 深 | 深 | **浅**（4-8） |
| 偏差/方差 | 高方差 | 降方差 | **降偏差** |

**主流实现**：LightGBM（微软，**首选**，比 XGBoost 快 10 倍）/ XGBoost / CatBoost。

## 5.2 为何选 LambdaRank

| 目标 | 回归（MSE） | LambdaRank |
|---|---|---|
| 学的是什么 | 涨幅绝对值 | 同日股票相对顺序 |
| 极端值敏感 | **极敏感** | 不敏感 |
| 与 Top-N 选股 | 弱相关 | **直接对齐** |
| 损失 | MSE | NDCG（截断 Top-N） |

**Top-N 选股本质是排序问题，不是预测问题。**

## 5.3 标准训练配置（参数选择）

| 参数 | 推荐值 | 说明 |
|---|---|---|
| `objective` | `lambdarank` | 排序学习 |
| `metric` | `ndcg` | 评估排序质量 |
| `ndcg_eval_at` | [2, 5, 10] | 与 Top-N 选股目标一致 |
| `lambdarank_truncation_level` | 20 | 只关心前 20 名 |
| `num_leaves` | 31 (调参范围 15-127) | 树复杂度 |
| `learning_rate` | 0.03 (调参 0.01-0.1) | 单棵树贡献权重 |
| `min_data_in_leaf` | 200 | 量化噪声大，叶子要厚 |
| `feature_fraction` | 0.8 | 每棵树随机用 80% 因子 |
| `bagging_fraction` | 0.8 | 每棵树随机用 80% 样本 |
| `bagging_freq` | 5 | 每 5 轮做一次 bagging |
| `lambda_l2` | 0.1 | L2 正则化强度 |
| `num_boost_round` | 500 (配合早停) | 总迭代数 |

## 5.4 Purged Walk-Forward 交叉验证

```
[─── train 2 年 ───][gap 5d][── val 6 月 ──]
[──── train 2.5 年 ────][gap 5d][── val 6 月 ──]
[───── train 3 年 ─────][gap 5d][── val ──]
```

- **embargo = 标签视界 + 1**（`fwd_5d_ret` → embargo = 6 个交易日；strategy-aware → max_hold + 1 = 21 个交易日）
- **不要用随机 K-fold**（金融数据有时间相关性，会严重高估性能）

## 5.5 Optuna 调参（4 个主旋钮）

调参对象：`num_leaves` / `learning_rate` / `min_data_in_leaf` / `feature_fraction` / `lambda_l2` / `max_depth`。

**预算建议**：100 trials 通常已收敛；超过 200 你在调"运气"。

**搜索策略**：TPE（Tree-structured Parzen Estimator）比 GridSearch 高效 10 倍，配合早停防过拟合到验证集。

## 5.6 反过拟合三件套

### 1. 单调性约束（业务先验比数据可靠）

为每个因子指定单调方向：
- PE 越低越好 → -1（负单调）
- 动量 → +1（正单调）
- U 型因子（如 `wave_rise_pct`） → 0（无约束，让模型自学）
- 触发型扣分（如 `wave_limit_up_low_vol`） → -1

加约束**几乎不损失训练精度，但显著提升 OOS 稳定性**——防止模型在某些区段学反。

### 2. 特征 / 样本比 ≥ 1:1000

41 因子 × 3 变换 ≈ 123 维 / 250 万样本 ≈ 20000，安全。
冷启动只有 1 年数据：5000 股 × 250 日 = 125 万行 / 123 列 ≈ 10000，仍可。

### 3. 早停 + Seed Averaging

- 早停：`num_boost_round=2000` 配合 `early_stopping(50)`，验证集 50 轮无提升停止
- Seed Averaging：训 5 次取平均（seed=42/123/456/789/1024），降低单次训练的随机性

## 5.7 三层评估指标

| 层 | 指标 | 判定门槛（A 股基准） |
|---|---|---|
| 因子层 | RankIC mean / IR | RankIC > 0.04, IR > 0.5 |
| 组合层 | Top-N 年化 / Sharpe / MDD | 年化 > 沪深 300 + 15%, Sharpe > 1.5 |
| 成本后 | 扣单边 0.15% 手续费 + 0.15% 冲击 | 年化仍 > 沪深 300 + 10% |
| 稳定性 | OOS / IS 性能比 | > 0.6 → 可信 |

## 5.8 三组对照实验（必跑）

| 实验 | 操作 | 期望 |
|---|---|---|
| 随机标签 | 把 forward_return 打乱再训 | IC 近似 0；不为零说明有泄露 |
| 随机因子 | 把因子替换为高斯白噪声 | 模型大幅劣化；不劣化说明因子贡献微弱 |
| 截止外推 | 仅用 2018-2020 训，2021-2023 测 | IC 下降幅度 < 50% 即稳定 |

## 5.9 部署：模型即数据

LightGBM 的杀手锏：**模型可序列化为纯文本 + 推理零依赖**。

- 训练完保存为 `.txt` 文本格式（可 diff），元数据（factor_cols 顺序、归一化参数、训练期、fingerprint）单独存 JSON
- 推理只需 LightGBM + numpy
- 全市场 5000 股推理 < 1 秒，单机绰绰有余

### 部署优势对照

| 维度 | LightGBM | TensorFlow / PyTorch |
|---|---|---|
| 推理依赖 | 1 个 .so 文件 | CUDA + Python + 框架 |
| 单股推理延迟 | < 0.1ms | 5-10ms |
| 模型大小 | 几 MB | 几十 MB - GB |
| 可解释性 | SHAP + 树可视化 | 黑箱 |

## 5.10 监控指标（每日盘后）

| 指标 | 触发阈值 | 动作 |
|---|---|---|
| 特征 PSI（分布漂移） | > 0.2 | 告警，检查行情结构变化 |
| 滚动 30 日 RankIC | 由正转负持续 2 周 | 该因子权重置零 |
| SHAP 重要性漂移（余弦距离） | > 0.3 | 模型可能退化，准备重训 |
| 模型整体 IR 衰减 | 较训练期下降 > 50% | 触发重训 |
