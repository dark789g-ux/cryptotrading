## 4.1 标签设计矩阵

| 标签 | 公式 | 适用 |
|---|---|---|
| `fwd_5d_ret` | 收盘价 5 日比例变化 | 最常用 |
| `fwd_5d_max_ret` | 5 日内最高价相对今日收盘 | 波段策略 |
| `fwd_5d_vwap_ret` | 5 日 VWAP 均值相对今日收盘 | **实盘对齐推荐** |
| `rank_in_day` | 日内 forward_ret 的 pct_rank | **LambdaRank 首选** |
| `is_top_decile` | rank_in_day >= 0.9 的二值 | 简单二分类 |
| `triple_barrier` | 先达到 +X% 或 -Y% 或时间到期 | López de Prado 方法 |
| `strategy_aware` | 按真实出场规则模拟交易 → 实现收益 | **主推**，见 4.2 |

**推荐组合**：
- **主标签**：`strategy_aware`（业内称 *realized return labeling*）的排序形式，与实盘 P&L 1:1 对齐
- **Baseline 对照**：`fwd_5d_vwap_ret`（fixed horizon），用于验证主标签的有效性
- 因子探索阶段：仍用 `fwd_5d_ret`，干净无策略干扰

## 4.2 Strategy-aware labeling 落地（推荐主用方案）

### 4.2.1 方案本质

用**真实交易出场规则**模拟生成标签：T 日收盘判断信号 → T+1 入场 → 按既定出场条件触发后用 VWAP 卖出 → 实现收益 = 标签。这是 López de Prado 在《Advances in Financial ML》第 3 章推荐的方法，业内称 *realized return labeling*。

固定窗口标签（`fwd_5d_ret`）的本质是"假装持有 5 天"——但实盘永远不会这样交易。strategy-aware 让训练目标和实盘目标 1:1 对齐。

### 4.2.2 与 fixed horizon 的关键差异

| 维度 | Fixed horizon (`fwd_5d_ret`) | Strategy-aware |
|---|---|---|
| 持有期 | 强制 5 天 | **自适应** 1-N 天 |
| 与实盘对齐 | 弱 | **强** |
| 是否含止损执行 | 否 | **是** |
| 信噪比 | 中 | **通常更高** |
| CV embargo 要求 | 6 个交易日 | **max_hold + 1** 个交易日 |
| 样本相关性 | 中 | **高**（重叠持仓窗口） |
| 实现复杂度 | 极简 | 中 |

### 4.2.3 推荐实现（单屏障 + 三道保险）

| 项 | 规则 |
|---|---|
| 入场判定 | T 日收盘判断触发条件，T+1 入场 |
| 入场价 | T+1 日 VWAP |
| 入场过滤 | T+1 涨停（开盘 ≥ 前收 × 1.097） → 该样本 `label = NaN` |
| 出场触发（任一即止） | (1) 收盘 < MA5 → 次日 VWAP 卖出<br>(2) 累计跌幅 ≤ -8% → 次日 VWAP 止损<br>(3) 持有满 20 个交易日 → 强制平仓 |
| 出场价 | 触发次日 VWAP |
| 出场过滤 | 触发次日跌停（开盘 ≤ 前收 × 0.903） → 顺延至能卖的下一日 |
| 停牌处理 | 持有期内停牌 → 等复牌当日 VWAP；若覆盖 max_hold，按复牌价兜底 |

### 4.2.4 必须处理的 5 个坑

| 坑 | 表现 | 修复 |
|---|---|---|
| 出场价用收盘 / 开盘瞬时价 | 实盘不可达 | 用次日 VWAP |
| 无最大持有期 | 标签视界爆炸，CV embargo 拉爆 | 强制 `max_hold = 20 个交易日` |
| 涨跌停 / 停牌不模拟 | 标签看似可成交，实盘卖不出 | 涨停封死顺延，停牌等复牌 |
| CV embargo 不足 | 训练集与验证集泄露 | embargo ≥ `max_hold + 1`（≥ 21 个交易日） |
| label 强右偏 | 模型主拟合中间值，忽视尾部 | 用排序标签（LambdaRank）代替回归 |

### 4.2.5 标签落库规范（`ml.labels` 表）

| 字段 | 类型 | 说明 |
|---|---|---|
| `symbol` | VARCHAR(10) | |
| `buy_date` | CHAR(8) | 入场日 YYYYMMDD |
| `buy_price` | NUMERIC | 入场 VWAP（含复权） |
| `sell_date` | CHAR(8) | 出场日 YYYYMMDD |
| `sell_price` | NUMERIC | 出场 VWAP（含复权） |
| `holding_days` | SMALLINT | 实际持有交易日数 |
| `exit_reason` | VARCHAR(20) | `below_ma5` / `stop_loss` / `time_limit` / `suspended` |
| `realized_return` | NUMERIC | 主标签：`sell_price / buy_price - 1` |
| `rank_in_day` | NUMERIC | 当日触发样本组内 pct_rank |
| `label_class` | SMALLINT | 5 分类：0=大亏 / 1=小亏 / 2=平 / 3=小赚 / 4=大赚 |
| `label_version` | VARCHAR(20) | 标签生成逻辑版本 |
| `created_at` | TIMESTAMPTZ | 应用项目时间规范 |

主键：`(symbol, buy_date, label_version)`，允许同一交易日的同一标的在不同 label_version 下共存。

### 4.2.6 标签健康度指标（生成后必跑）

| 指标 | 期望区间 | 不满足时 |
|---|---|---|
| 平均 holding_days | 5-15 个交易日 | < 3 = MA5 太敏感；> 18 = 趋向 max_hold |
| 出场原因分布 | below_ma5 / stop_loss / time_limit ≈ 60 / 25 / 15 | 严重失衡需调阈值 |
| `realized_return` 偏度 | \|skewness\| < 1.5 | > 1.5 强烈推荐用排序标签 |
| 同股相邻样本 label autocorr (lag=1) | < 0.5 | > 0.7 说明 embargo 不够 |
| 入场过滤导致 NaN 比例 | < 15% | > 15% 检查涨跌停 / 停牌过滤是否过严 |
| 平均 `realized_return` | -2% ~ +5% | 全为负说明策略本身不成立，gate 选错维度 |

### 4.2.7 与 fixed horizon 的对照实验（必跑）

评估阶段并行训两版模型，**结果对照是验证主标签是否成立的最强证据**：

| 版本 | 标签 | 用途 |
|---|---|---|
| v1（baseline） | `fwd_5d_ret` 排序版 | 控制组 |
| v2（主推） | strategy-aware 排序版 | 实验组 |

**判定标准**：
- v2 OOS Sharpe 比 v1 高 20%+ → strategy-aware 在当前策略下有效，主推
- v2 OOS Sharpe 比 v1 低 → 策略出场规则可能与 alpha 不匹配，回退 fixed horizon 并重审策略
- 两者接近 → 出场规则尚未提供额外 alpha，但保留 strategy-aware 以便后续止盈止损扩展

### 4.2.8 何时**不要**用 strategy-aware

| 场景 | 用什么 | 原因 |
|---|---|---|
| 因子研究阶段（单因子 IC） | `fwd_5d_ret` | 干净无策略干扰，方便横向比较 |
| 多策略并行的母模型 | `fwd_5d_ret` | 不同策略卖出规则不同，标签不统一 |
| 论文 / 学术对标 | `fwd_5d_ret` | 业界标准对比基准 |
| 单策略上线版 | **strategy-aware** | 与实盘 1:1 对齐 |

## 4.3 标签的污染过滤（生成时必做）

| 污染源 | 表现 | 修复 |
|---|---|---|
| T+1 涨停 | 实盘买不到，训练却把高涨幅算成胜 | 排除"次日开盘价 ≥ 今日收盘价 × 1.097"的样本 |
| T+horizon 跌停 | 卖不出 | 排除"末日开盘价 ≤ 前日收盘价 × 0.903"的样本 |
| 停牌跨日 | forward 涨幅为 0 但不可交易 | 标记 `is_suspended` 期间样本为 NaN |
| 新股 | 上市 60 日内噪声大 | 直接从训练集排除 |
| 退市股 | 最后交易日之后不能 forward | 跨越 `delist_date` 的样本置 NaN |

## 4.4 标签的"四种污染"检测

| 污染类型 | 表现 | 检测方法 | 修复 |
|---|---|---|---|
| 未来数据泄露 | OOS IC 巨大暴跌 | shift 检查 + PIT 复现 | 严格 forward 计算 |
| 存活偏差 | 训练集只含未退市股 | 训练集股票数 vs 当时上市股数 | 用 PIT universe |
| 涨跌停不可交易 | 训练 IC 高但实盘买不到 | 检查买入日是否涨停 | 排除涨停信号 |
| 类别不平衡 | top 标签太稀疏 | 各 label 计数 | 调权重 / 用排序 |
