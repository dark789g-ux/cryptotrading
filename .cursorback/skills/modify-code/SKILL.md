# 代码修改工作流

在修改任何代码前、中、后执行的标准化流程，确保 AGENTS.md 文档始终与代码保持同步。

## Step 1: 定位 AGENTS.md 文档

1. **检查用户是否指定了 AGENTS.md 路径**
   - 如果用户提供了具体路径，使用该路径
   - 否则，进入下一步

2. **查找当前目录下的 AGENTS.md**
   - 检查用户请求涉及的文件所在目录
   - 如果存在 AGENTS.md，读取并理解项目结构

3. **处理数据目录**
   - 如果目录下**全部是数据文件**（.csv, .json, .parquet 等）：
     - 跳过在该目录创建 AGENTS.md
     - 向上级目录查找 AGENTS.md
     - 确保上级目录的 AGENTS.md 中说明数据结构

4. **处理 IDE 配置目录**
   - 以下目录**不创建** AGENTS.md：
     - `.cursor/` — Cursor 编辑器配置
     - `.vscode/` — VS Code 编辑器配置
   - 这些目录的说明应在**项目根目录**的 AGENTS.md 中统一记录

## Step 2: 创建缺失的 AGENTS.md（如需要）

如果相关目录（非纯数据目录）不存在 AGENTS.md，使用标准模板创建。

**模板位置**: [references/agents-template.md](references/agents-template.md)

创建时：
1. 读取模板文件
2. 替换 `[目录名称]` 为实际目录名
3. 根据目录内容填充文件索引和约束

## Step 3: 执行代码修改

根据 AGENTS.md 中的约束和用户需求修改代码：

1. **遵循项目约定**:
   - 所有 `.py` 文件首行 `# -*- coding: utf-8 -*-`
   - 配置参数以大写常量声明在文件顶部
   - 使用 pandas 处理 CSV
   - 使用 fastapi 构建后端服务
   - 根据设备核数设置 `MAX_WORKERS`

2. **保持风格一致**:
   - 遵循现有代码的命名规范
   - 使用项目中已有的工具库
   - 遵循 AGENTS.md 中定义的关键不变量

## Step 4: 更新 AGENTS.md

修改完成后，更新相关 AGENTS.md：

1. **更新文件索引**: 如有新增/删除/重命名文件
2. **更新不变量**: 如有新的约束条件需要添加
3. **更新数据结构**: 如涉及数据文件格式变更


## 工作流检查清单

```
- [ ] 如果不存在且非纯数据目录，创建标准结构

- [ ] 更新相关不变量（如需要）
```

## 特殊情况处理

### 跨目录修改
如果修改涉及多个目录，为每个目录重复上述流程。

### 创建新目录
1. 创建目录
2. 在该目录创建 AGENTS.md
3. 在父目录的 AGENTS.md 中添加索引引用

### 纯数据目录判定标准
以下情况视为**纯数据目录**（不创建 AGENTS.md）：
- 文件 90% 以上是数据文件（.csv, .json, .parquet, .db, .sqlite）
- 无 Python/JS/TS 等源代码文件
- 无配置文件（.yaml, .toml, .ini）

对于纯数据目录，必须在其**上级目录**的 AGENTS.md 中说明数据结构。
