## 第一步：提取三类知识

从对话中识别以下内容，**只提取有实际价值的信息，跳过显而易见的常识**：

**A. 用户偏好**
- 代码风格、命名规范、文件/目录结构
- 技术选型（框架、库、工具链）
- 工作流程、操作习惯
- 对 AI 辅助方式的偏好（如：喜欢先看方案再实现、不喜欢过度注释等）

**B. 重要约束**
- 必须遵守的架构规则、接口约定
- 明确禁止使用的方案或模式（及原因）
- 性能、安全、兼容性等非功能要求
- 团队协作规范

**C. 踩坑记录**
- 本次走过的弯路、错误方向
- 第三方库 / API / 工具的非预期行为
- 环境配置、依赖、版本兼容问题
- 任何让开发卡住超过 10 分钟的问题

---

## 第二步：生成 doc/ 细节文档

对第一步提取的每一条信息，判断是否需要独立的细节文档：

**需要单独文档的条件**（满足其一即创建）：
- 解释超过 3 行
- 包含代码示例、配置片段或操作步骤
- 是踩坑记录（踩坑一律建文档）

**文档规范：**
- 路径：`doc/<英文短横线命名>.md`（如 `doc/api-retry-pitfall.md`）
- 命名：用动词或名词短语描述主题，如 `auth-flow`、`avoid-axios-interceptor`
- 内容结构：
  ```
  # <标题>
  
  ## 背景
  <为什么这条规则/踩坑存在>
  
  ## 结论
  <一句话核心结论>
  
  ## 详情
  <代码示例、步骤、原因分析>
  ```
- 若文件已存在：追加新内容（加分隔线 `---` 和日期），不覆盖旧内容

## 第三步：更新 AGENTS.md

**若 AGENTS.md 已存在**：
1. 读取现有内容
2. 将新提取内容与现有内容合并，去除重复项
3. 删除明显过时或已被新内容覆盖的条目
4. 严格控制总行数在 50–100 行

**若 AGENTS.md 不存在**：新建文件。

**AGENTS.md 格式模板（严格遵守）：**

```markdown
# AGENTS.md

> 供 AI 助手读取。记录团队偏好、约束与踩坑经验，细节见 `doc/`。
> 最后更新：{YYYY-MM-DD}

## 偏好

- {一句话描述} [→ 详见](doc/xxx.md)
- {一句话描述}

## 约束

- **禁止**：{禁止的做法} — {一句话原因} [→ 详见](doc/xxx.md)
- **必须**：{必须的做法} — {一句话原因}

## 踩坑记录

- **{坑的名称}**：{一句话描述现象和解法} [→ 详见](doc/xxx.md)
```

**写入规则：**
- 每条记录不超过 1 行
- 有细节文档的条目必须附链接，无需在此展开
- 不写废话、不写通用最佳实践（如"写注释"、"做错误处理"）
- 相同主题的多条合并为一条
- 语言：中文
