## 一、跨子系统系统性问题（最高优先级）

以下问题相互关联，会**系统性污染训练数据与评估指标**，且都不报警。在用本管线的任何指标做模型决策前必须先修。

### 🔴 A. 标签收益率从未做后复权 —— 污染所有标签
`labels/runner.py:49-71` 的 `_load_daily_quotes` 直接 `SELECT q.close FROM raw.daily_quote`，从不 JOIN `raw.adj_factor`。但 `fallback.py:13`、`strategy_aware.py:14`、`exit_rules.py:60` 三处文档都写「后复权 close」。后果：持有期内发生分红/送转的票，除权缺口被当成真实收益写进 `factors.labels`，还会误触 -8% 止损。**确凿正确性 Bug，最先修。** 详见 [03-labels.md](03-labels.md) 问题 1。

### 🔴 B. 交易成本被双重扣除
`labels/strategy_aware.py:443` 的 label 已是净收益（`gross - ROUND_TRIP_COST`）；`evaluation/portfolio.py:136` 又按 `turnover × (commission + slippage)` 再扣一次。需先对齐口径再决定哪一层扣。详见 [05-inference-evaluation.md](05-inference-evaluation.md) 问题 1。

### 🔴 C. label 持仓 horizon 语义全管线认知不一致
`evaluation/portfolio.py` 顶部说「持有 1~20 日累计收益」，同文件 docstring 又说「持有到下个交易日」。这一矛盾连带使 Sharpe 年化、`equity` 复利、NDCG gain 口径三处指标不可信。根因是 label 定义无统一文档约定。详见 [05-inference-evaluation.md](05-inference-evaluation.md) 问题 2。

### 🔴 D. 标签在训练通路被双重变换
`training/runner.py:326` 先 `_bin_labels_by_group` 把连续 label 压成 0~4 五档，`ab_compare.py:229` 又做截面 rank。导致回归目标被压成台阶、LambdaRank tie 随机打破、IC/NDCG/portfolio 评估全部失真。详见 [04-training.md](04-training.md) 问题 1。

### 🔴 E. 整个交易日数据缺失能骗过质量门禁
`quality/checks_value.py` 的 `null_violation`、`cross_table_alignment` 在「当日表 0 行」时静默判 `passed=True`；`quality/runner.py:84` 在 `strict=False` 时把崩溃的检查包装成 `CheckResult(passed=True)`。某交易日 raw 没同步时门禁仍判绿。详见 [06-quality.md](06-quality.md) 问题 1-3。

### 🔴 F. PIT 审计实质失效（给出误导性绿灯）
`quality/pit_audit.py` 的 `audit_ghost3_fina_delay`、`audit_ghost2_adj_trap` 既假阳又假阴，不具审计效力，却挂着绿灯让人误信「PIT 已审计」。应明确标注「未实现」。详见 [06-quality.md](06-quality.md) 问题 4-5。

## 二、各子系统其余 🔴 严重问题速览

- **sync**：客户端 bug 被吞进重试伪装成「无数据」；无日期接口伪造 `quality_reports.trade_date`；`upsert_rows` 不校验 row key 一致性。
- **factors/features**：整列因子算不出来时被静默填 0 进训练矩阵，不报警、不 drop。
- **training**：单 fold 通路调用未定义的 `_progress` 必崩；该通路早停在 test 段偷看（泄漏）；Purged WF 测试窗口只覆盖数据后半段。
- **inference**：`_load_model_run` 用 `LIMIT 1` 无 `ORDER BY`，同名 model_version 多行时可能加载错模型。
- **worker**：runner 主动失败的 job 永不重试；长任务缺 heartbeat 线程，会被 reaper 误杀并并发跑两遍。

## 三、建议修复顺序

1. **A** — 标签后复权（影响面最大、修复明确）
2. **B / C** — 成本与 horizon 口径（需团队对齐定义）
3. **E** — 质量门禁空表漏报（防止后续问题被掩盖）
4. **D** — 标签双重变换
5. **worker** — 失败重试 / heartbeat
6. 其余按各子系统文档内优先级处理

前五条不修，本管线产出的任何回测指标都不可信。

## 四、整体评价

架构骨架扎实——PIT 框架有清醒注释、空数据三路分流、upsert 去重、worker 快乐路径干净、artifact 回滚都到位。但本次评审暴露一个共同模式：**「声明的契约」与「实际实现」系统性脱节**——文档说后复权实际没复权、文档说 walk-forward 实际只测后段、PIT 审计挂绿灯实际不具效力、检查崩溃却报 passed。这类问题比单纯 bug 更危险，因为它们让残缺/失真的数据带着「已验证」的假象流向模型决策。
