# 03 · 新建回测 UI 与配置页下线

## 信息架构

```text
现状
  Sidebar: [Regime 回测] [Regime 配置]
  新建 → 选配置版本 + capital 仓位

目标
  Sidebar: [Regime 回测]
  新建 → 内嵌象限编辑 + 资金/区间 → 新建并运行
  /regime-config → 重定向 /regime-backtest
```

## 新建弹窗 wireframe

```text
┌─ 新建 Regime 回测（宽 ~900px）─────────────────────────────┐
│  ── 基本信息 ──                                             │
│  方案名          [____________]                             │
│  回测区间        [起始] — [结束]                            │
│                                                             │
│  ── 资金 ──                                                 │
│  初始资金        [1,000,000]                                │
│  成本预设        [现实 ▼]                                   │
│  （无：仓位比例、最大持仓、象限覆盖提示）                     │
│                                                             │
│  ── Regime 规则（内嵌 RegimeConfigEditor）──              │
│  （内嵌时隐藏版本号与备注字段；回测不展示配置元数据）       │
│  [+ 添加象限]                                               │
│  ┌ tabs ──────────────────────────────────────────────────┐ │
│  │ 标签 / 动作 trade|flat                                 │ │
│  │ trade → 仓位比例* / 最大持仓* / 分桶 / 入场 / 出场     │ │
│  │ flat  → 无仓位、无入场出场                             │ │
│  └────────────────────────────────────────────────────────┘ │
│                                                             │
│                         [取消]  [新建并运行]                │
└─────────────────────────────────────────────────────────────┘
```

## 页面文案

- 副标题改为：「配置 Regime 规则并运行组合回测」  
- 详情抽屉：增加可折叠「本次规则摘要」（含各象限仓位与出场模式）

## 配置页下线

| 项 | 行为 |
|----|------|
| Sidebar | 移除「Regime 配置」 |
| 路由 | `/regime-config` redirect → `/regime-backtest` |
| 后端 CRUD / activate | 一期可保留（脚本）；前端不再暴露 |

## 象限仓位 UI

- `trade`：仓位比例、最大持仓为必填（不再「可选 / null」）  
- 保存前校验 `positionRatio × maxPositions ≤ 1`  
- 与 [04-trailing-lock-params-ui.md](./04-trailing-lock-params-ui.md) 同属编辑器改动
