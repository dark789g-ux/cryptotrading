## 4.3 响应式空间分析

核心矛盾：侧栏展开后 K 线会被挤压，而详情面板宽度取决于 split-pane 列表占比（列表可拖到 60%，详情只剩 40%）。

```text
列表占比 40%（默认）→ 详情 ≈ 60% 宽
  1440px 屏: 详情 ~864px → 展开(侧栏260) 后 K线 ~604px ✓
  1024px 屏: 详情 ~614px → 展开后 K线 ~354px ⚠ 接近极限
列表占比 60%（用户拖宽列表）→ 详情 ≈ 40% 宽
  1440px 屏: 详情 ~576px → 展开后 K线 ~316px ✗ K线过窄不可用
```

守卫逻辑（详见 `./02-architecture.md#23-响应式守卫`）：容器可用宽度 < 620px 时自动折叠 + 禁用按钮。

## 4.4 错误处理

| 场景 | 处理 |
|---|---|
| `row` 为 null | `*InfoFields` 内部 `v-if="row"` 守卫字段列表不渲染，组件渲染 `<n-empty description="未选择标的" size="small" />` 占位；侧栏框架始终存在，body 区显示空状态文案 |
| 单字段 null/undefined | formatter 内置返回 `'-'` |
| 美股/加密某行情字段缺失 | 同上，显示 `'-'` |
| K 线未加载完 | 面板独立于 K 线加载状态，数据来自 `row` 快照，不受影响 |
| 容器宽度 < 620px | 按钮禁用 + tooltip"空间不足"；若已展开则自动折叠 |
