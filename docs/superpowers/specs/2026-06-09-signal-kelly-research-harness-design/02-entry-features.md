# 02 · 新入场特征库（撬动胜率 p / 自然减信号）

← 返回 [index](./index.md)　|　上一篇 [01 架构](./01-architecture-dataflow.md)　|　下一篇 [03 出场](./03-exit-structures.md)

入场变体 = 在「base 触发」之上 **AND 叠加** 若干特征阈值。特征都在 `signal_date` 截面计算，落成信号宽表的列；扫描时对列加掩码即得变体，不重算路径。

## 1. 特征定义表

| 特征 | 列名 | 定义 / 公式 | 数据源 | 候选阈值（生成变体） |
|---|---|---|---|---|
| base 超卖 | `kdj_j` | 信号日 KDJ_J 值 | `raw.daily_indicator.kdj_j` | `<0` / `<-10` / `<-20` |
| 超跌幅度 | `dev_ma5` | `qfq_close / ma5 − 1` | quote.qfq_close + indicator.ma5 | `<-3%` / `<-5%` / `<-8%` |
| 超跌幅度(中期) | `dev_ma30` | `qfq_close / ma30 − 1` | quote + indicator.ma30 | `<-8%` / `<-12%` |
| 连阴天数 | `down_streak` | 截至信号日连续 `qfq_pct_chg<0` 的天数 | `raw.daily_quote.qfq_pct_chg` | `>=3` / `>=4` / `>=5` |
| 缩量比 | `vol_contract` | `vol / mean(vol, 过去5可交易日)` | `raw.daily_quote.vol` | `<0.7` / `<0.5` |
| 波动区制 | `vol_regime` | `atr_14 / qfq_close` 的全市场当日分位 | `indicator.atr_14` + quote | 低波动档（如分位 <0.3） |
| 相对强弱 | `rs_vs_index` | 个股 N 日尾盘收益 − 同期基准收益 | quote + ths_index_daily_quotes | `>0`（强于基准） / `<0`（跟跌） |

> **口径警告**：`dev_ma*` / `vol_regime` 依赖 `ma*` / `atr_14` 的复权口径。实现前必须确认指标表是基于复权价还是原始价（[index §7 第1条](./index.md#7-实现前需落源头确认的项不得凭推断写死)），否则偏移系统性失真。

## 2. 变体枚举策略

- **base 取宽**：信号枚举（[01 ①](./01-architecture-dataflow.md#1-phase-1-数据流六段-pipeline)）用最宽的 base（如 `KDJ_J<0`），一次加载覆盖全部更严变体的路径。
- **变体 = base + 0~2 个附加特征阈值**的 AND 组合。控制组合爆炸：默认每个变体最多叠加 **2** 个附加特征（可配 `max_entry_filters`），避免维度灾难 + 过拟合。
- 每个特征的阈值候选见上表；变体集合 = base 阈值 × 附加特征阈值的笛卡尔积（受 `max_entry_filters` 截断）。
- 变体的信号数随阈值收紧单调下降——这正是"少信号"的来源；凯利是否随之上升由扫描回答。

## 3. RS（相对强弱）专项

RS 是唯一依赖外部基准、且有数据时间约束的特征，单列说明。

### 3.1 口径

`rs_vs_index(N) = 个股[signal_date 起回看 N 个可交易日的收益] − 基准[同期收益]`。

- N 默认 5（可配 `rs_lookback`，候选 5 / 20）。
- 个股收益用 `qfq_close`；基准收益用 `ths_index_daily_quotes.close`（指数无需复权）。
- 含义：在"超跌"前提下区分**超跌但强于基准（抗跌）** vs **跟跌（弱势）**，分别作为两类变体（`rs>0` / `rs<0`）测试，不预设哪种更优——由扫描定。

### 3.2 基准（运行时可配，默认沪深300）

| 基准 | 代码 / 来源 | 含义 |
|---|---|---|
| 沪深300样本股（默认） | `883300.TI` | 大盘相对强弱 |
| 中证500成份股 | `883304.TI` | 中盘相对强弱（超跌反弹多在中小盘，可能更贴样本性质） |
| 所属行业指数 | type I + `ths_member_stocks`(con_code→ts_code) 映射 | 行业内相对强弱；个股映射到多个 type I 行业指数时，**默认取成份股数最多者（最广义行业），并列则按 ts_code 升序取第一**，保证可复现 |

配置项 `rs_benchmark ∈ {hs300, zz500, industry}`，可多选（多选则每个基准各生成一组 RS 变体）。

### 3.3 数据时间约束（硬约束）

THS 指数日线**仅 2024-01-02 起**（[index §6](./index.md#6-已核对的事实锚点实现时直接信任禁止再凭二手转述改写)）。因此：

- **含 RS 的入场变体，其评估窗口自动夹到 `signal_date >= 2024-01-02`**；当 RS 变体启用且配置的训练起点早于 2024-01-02 时，训练起点 **clamp 到 2024-01-02** 并在报告中显式标注（不得静默丢 2023 信号——符合 `data-integrity.md`「外部空数据/降级须 warn」精神）。
- **不含 RS 的变体仍用完整 2023–2026 窗口**，与含 RS 变体在报告中分区呈现，避免跨不同样本窗口直接比凯利造成误导。
- 行业基准还受 `ths_member_stocks` 覆盖约束：映射缺失的个股，其 RS 变体样本应被显式标记为"基准缺失"而非按 0 处理。
