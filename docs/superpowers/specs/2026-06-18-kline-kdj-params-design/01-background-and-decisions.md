# 1. 背景与关键决策

## 1.1 背景

`KlineChartToolbar.vue` 当前用一颗总齿轮管理所有副图的显隐、高度与排序，每个指标没有独立参数设置入口。KDJ 在后端/DB 中默认按 `9/3/3` 预计算，无法在前端调整。

**目标**：在 KDJ 指标名称后添加齿轮按钮，点击后可在行内 Popover 修改 `N/M1/M2`，点确定后触发后端按新参数重算 KDJ 并刷新 K 线图。

## 1.2 关键决策

| 问题 | 决策 |
|---|---|
| 支持哪些指标 | 仅 KDJ |
| 参数范围 | `n: 2–99`，`m1/m2: 1–50`，默认 `9/3/3` |
| 指标名显示 | 有自定义参数时显示 `KDJ(n,m1,m2)`，否则显示 `KDJ` |
| 设置面板 | 行内 `NPopover`，内嵌三个 `NInputNumber` + 确定/取消 |
| 重算方式 | 方案 1：前端改参后，立即调用后端 recalc 接口，返回完整 bars 并替换 |
| 失败处理 | 后端失败时回滚参数到上一版成功值/默认值，并 toast 提示 |
| 持久化 | 参数随 `SubplotPrefs` 一起存 `localStorage`（按 `prefsKey` 隔离） |
| 本期接入范围 | Crypto 详情页 + A 股详情页；其它入口暂不接入后端 recalc |

## 1.3 为什么只接 Crypto + A 股

- 两者已有现成的 K 线查询接口和数据表结构，新增 recalc 端点改动最小；
- 美股、美股指数、回测的数据源/表结构不同，需要各自独立实现重算逻辑，会显著扩大本期范围；
- KlineChart 组件本身保持通用：通过可选 prop `recalcIndicators` 接入，未传入的页面仅显示齿轮并持久化参数，不自动刷新。
